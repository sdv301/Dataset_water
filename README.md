# HydroPredict: Система вероятностного прогноза паводков

Добро пожаловать в проект **HydroPredict**. Эта система состоит из ML-бэкенда (Python) и современного интерфейса для визуализации данных и дашбордов (React + Vite).

Система позволяет строить вероятностные прогнозы уровней рек, управлять виджетами аналитики, смотреть на влияние предикторов, а также имеет интерфейс для загрузки новых данных и инкрементального переобучения модели.

---

## 🛠 Архитектура

Проект разделен на две логические составляющие:

### 1. Frontend (React + Vite, текущая директория)
- **Фреймворк**: React 18, Vite.
- **Стилизация**: Tailwind CSS (утилитарный подход).
- **Визуализация**: `recharts` для интерактивных графиков, `react-leaflet` для работы с геоинформационными картами.
- **Иконки**: `lucide-react`.
- **Основной код**: `src/App.tsx`. Здесь реализована логика управления состоянием дашбордов (включение/отключение виджетов), вкладка загрузки данных и рендеринг картографической информации.

### 2. Backend / ML Prototype (Директория `/python_code/`)
- **Скрипт**: `flood_predictor.py`
- **Алгоритм**: Ансамблевое прогнозирование на основе градиентного бустинга (`XGBRegressor`). Обучается на кастомном loss'е (Pinball loss) для получения перцентилей (медиана 0.5, пессимистичный 0.95, и т.д.). Трекинг гиперпараметров осуществляется через `Optuna`. Скользящая кросс-валидация через `TimeSeriesSplit`.

## 🚀 Как запустить Frontend (Дашборды)

Если вы работаете локально или в AI Studio:

1. Установите зависимости (если они еще не установлены):
   ```bash
   npm install
   ```
2. Поднимите Development-сервер:
   ```bash
   npm run dev
   ```
3. Откройте интерфейс: `http://localhost:3547`.

### Что реализовано в интерфейсе?
- **Главный экран (Прогноз на дату)**: Вероятностный гидрограф, где отображена Базовая линия (тренд) и вероятностные границы будущих уровней реки.
- **Дашборды (Аналитика)**: Включая возможность самостоятельного включения/отключения виджетов (Сравнение моделей, Корреляция аномалий (Scatter plot), Риск по бассейнам, Матрица корреляций (Heatmap) и Важность признаков).
- **Управление данными**: Экран для загрузки новых `.csv` наборов данных и кнопки "Инкрементальное обучение" / "Полный ретрейн" для интеграции с пайплайном.
- **Карта**: Интерактивная GIS карта построенная на Leaflet с точками гидропостов и цветовой кодировкой риска.

## 🐍 Подключение Backend (Python)

Так как текущая система запущена в node/браузерном окружении, чтобы "оживить" датасеты и кнопки ретрейна, вы можете обернуть Python скрипт в FastAPI.

### 1. Установка Python зависимостей
```bash
pip install -r python_code/requirements.txt
# включает CatBoost (по умолчанию), XGBoost, Optuna, FastAPI
```

### 2. Запуск пайплайна локально
Вы можете запустить пример обучения (если вы создадите датасет):
```python
from flood_predictor import FloodPredictor
import pandas as pd

# Создание инстанса модели
predictor = FloodPredictor()

# Если у вас есть df (с колонками date, temp_mean_c, precip_mm, snow_depth_cm, water_level_cm)
# predictor.train(df, target_col="water_level_cm")
```

### Связывание Frontend с Python API
Чтобы кнопки "Ретрейн" заработали с реальным бэкендом, создайте API эндпоинты в `FastAPI` (например: `POST /api/train`) и вызовите их в реагирующем приложении с помощью обычного `fetch('http://localhost:8000/api/train', { method: 'POST' ... })`. В данный момент в интерфейсе заложены моки для плавного визуального восприятия.
