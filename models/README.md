# Сохранённые ML-модели

Обученные модели лежат здесь: **`models/{река}/{пост}/`**

| Файл | Назначение |
|------|------------|
| `model_h*_q*.joblib` | CatBoost/XGBoost по горизонту и квантилю |
| `manifest.json` | Дата обучения, backend, метрики |
| `features.joblib` | Список признаков |

Метаданные дублируются в SQLite (`station_models` в `data/ml_features.db`).  
**Сама БД в Git не кладётся** (см. `.gitignore`), только при необходимости — выбранные папки моделей.

## Одна станция — обучить один раз

```powershell
cd python_code
.\.venv\Scripts\python train_station.py --river "Лена" --post "Якутск"
```

Или в интерфейсе: **Управление данными → Быстрое (~5–15 мин, 1 станция)**.

После обучения API **подхватывает файлы с диска** — переобучать при каждом запуске не нужно.

## Закоммитить модель в GitHub

Только нужную станцию (не всю папку `models/`, если там много постов):

```powershell
git add models/Лена/Якутск/
git commit -m "Модель CatBoost: Лена — Якутск"
git push
```

На другом компьютере: `git clone` → `npm run start` — прогноз сразу с диска.

**Ограничение GitHub:** файл до 100 MB, репозиторий — разумный размер.  
Одна станция (~20 `.joblib`) обычно 5–50 MB. Если больше — используйте [Git LFS](https://git-lfs.github.com/) для `*.joblib`.

## Когда переобучать

- Загрузили новые наблюдения в БД (`prepare_ml_data.py` / загрузка CSV)
- Сменили признаки или код `flood_predictor.py`

Иначе старая модель в `models/` остаётся актуальной.
